# RLS (Row-Level Security) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> Spec di riferimento: [`docs/superpowers/specs/2026-06-24-rls-design.md`](../specs/2026-06-24-rls-design.md).

**Goal:** Chiudere gli ultimi due buchi di autorizzazione lato DB così da poter aprire l'app al pubblico in sicurezza.

**Architecture:** **L'audit (Step 0) ha rivelato che le RLS sono già implementate e corrette su quasi tutte le tabelle** (helper `is_campaign_member`/`is_campaign_master` già presenti, FK già `ON DELETE CASCADE`, policy coerenti con lo spec). La descrizione "RLS permissive" in DA-FARE era **superata**. Restano **due gap**: (A) `combat_state` ha una policy `ALL true/true` (spalancata); (B) `campaign_members_insert` permette l'auto-promozione a master e l'auto-join. Si chiude B con la variante completa: RPC `join_campaign` (definer) + policy di insert ristretta + modifica a `JoinCampaignAsync`. Rilascio ordinato: **RPC additiva → deploy client → lockdown SQL**.

**Tech Stack:** Supabase (PostgreSQL + PostgREST + Gotrue), Blazor WebAssembly .NET 10, `supabase-csharp`, deploy GitHub Pages via GitHub Actions.

### Task 3 — RPC `join_campaign` (additivo)

Funzione di join server-side che valida il codice e garantisce la membership `player`, bypassando la RLS. **Additiva**: il client attuale non la usa ancora; va creata **prima** del deploy del nuovo client (Task 4) e **prima** del lockdown (Task 5).

**Files:** Create `db/20-join-campaign.sql`

**Interfaces:**
- Consumes: `UNIQUE(campaign_id,user_id)` (già presente) per `on conflict`.
- Produces: `public.join_campaign(p_code text) → uuid` (id campagna se codice valido + membership `player` garantita; `null` se codice invalido). Consumato dal C# in Task 4.

- [ ] **Step 1: Creare `db/20-join-campaign.sql`**

```sql
create or replace function public.join_campaign(p_code text)
returns uuid language plpgsql security definer set search_path = public as $$
declare v_campaign uuid;
begin
  select id into v_campaign from campaigns where invite_code = upper(trim(p_code));
  if v_campaign is null then return null; end if;           -- codice non valido
  insert into campaign_members (campaign_id, user_id, role, joined_at)
  values (v_campaign, auth.uid(), 'player', now())
  on conflict (campaign_id, user_id) do nothing;            -- già membro → no-op
  return v_campaign;
end; $$;
```

### Task 4 — Client: `JoinCampaignAsync` → `join_campaign`

Va deployato **prima** del lockdown (Task 5), altrimenti la nuova policy bloccherà il vecchio INSERT diretto.

**Files:** Modify `Services/SupabaseService.cs:456-493` (`JoinCampaignAsync`)

**Interfaces:**
- Consumes: `public.join_campaign(p_code text) → uuid` (Task 3).
- Produces: firma invariata verso i chiamanti (`JoinCampaignAsync(string, string) → Campaign?`).

- [ ] **Step 1: Sostituire il corpo di `JoinCampaignAsync`**

```csharp
public async Task<Campaign?> JoinCampaignAsync(string inviteCode, string userId)
{
    var client = await GetClientAsync();
    var code = inviteCode.Trim().ToUpperInvariant();

    // join_campaign (SECURITY DEFINER) valida il codice e garantisce la membership "player"
    // server-side, bypassando la RLS. Ritorna l'id campagna o null se il codice non è valido.
    // Sostituisce find_campaign_by_invite_code + INSERT diretto, che la nuova policy
    // campaign_members_insert (solo owner-come-master) bloccherebbe.
    var campaignId = await client.Rpc<string>(
        "join_campaign",
        new Dictionary<string, object> { { "p_code", code } });

    if (string.IsNullOrEmpty(campaignId)) return null; // codice non valido

    var found = await client.From<Campaign>()
        .Where(c => c.Id == campaignId)
        .Get();
    return found.Models.FirstOrDefault();
}
```

### Task 5 — Lockdown: fix `combat_state` + `campaign_members_insert`

Chiude i due gap in **una transazione**. Da applicare **dopo** che il client nuovo è live.

**Files:** Create `db/30-rls-fixes.sql`, Create `db/99-rollback.sql`

**Interfaces:**
- Consumes: `is_campaign_member`/`is_campaign_master` (già presenti); `JoinCampaignAsync` via RPC già deployato (Task 4).
- Produces: `combat_state` scrivibile solo dal master, leggibile dai membri; `campaign_members` inseribile solo come owner-come-master (i player passano dalla RPC).

- [ ] **Step 1: Creare `db/99-rollback.sql` (rollback mirato, da tenere a portata)**

```sql
-- Ripristina le DUE policy precedenti (stato pre-fix) in caso di blocco.
drop policy if exists combat_state_select on combat_state;
drop policy if exists combat_state_insert on combat_state;
drop policy if exists combat_state_update on combat_state;
create policy "combat_state access" on combat_state for all using (true) with check (true);

drop policy if exists campaign_members_insert on campaign_members;
create policy campaign_members_insert on campaign_members for insert
  with check ( (user_id = auth.uid()) or is_campaign_master(campaign_id) );
```

- [ ] **Step 2: Creare `db/30-rls-fixes.sql`**

```sql
begin;

-- Gap A: combat_state (sostituisce la policy ALL true/true)
drop policy if exists "combat_state access" on combat_state;
create policy combat_state_select on combat_state for select using ( is_campaign_member(campaign_id) );
create policy combat_state_insert on combat_state for insert with check ( is_campaign_master(campaign_id) );
create policy combat_state_update on combat_state for update using ( is_campaign_master(campaign_id) )
                                                            with check ( is_campaign_master(campaign_id) );

-- Gap B: campaign_members_insert (solo owner-come-master; i player passano da join_campaign)
drop policy if exists campaign_members_insert on campaign_members;
create policy campaign_members_insert on campaign_members for insert
  with check ( user_id = auth.uid() and role = 'master'
               and exists (select 1 from campaigns c where c.id = campaign_id and c.owner_id = auth.uid()) );

commit;
```
