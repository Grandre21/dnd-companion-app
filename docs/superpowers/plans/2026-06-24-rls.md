# RLS (Row-Level Security) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.
>
> Spec di riferimento: [`docs/superpowers/specs/2026-06-24-rls-design.md`](../specs/2026-06-24-rls-design.md).

**Goal:** Spostare l'autorizzazione sui dati dal client al DB Supabase con policy RLS per ogni tabella, replicando esattamente il comportamento attuale, così da poter aprire l'app al pubblico in sicurezza.

**Architecture:** Funzioni `SECURITY DEFINER` (`is_campaign_member`, `is_campaign_master`, `can_read_character`, `can_edit_character`) centralizzano la logica ed evitano la ricorsione su `campaign_members`. Il join campagna passa a una RPC `join_campaign` (definer) per chiudere l'auto-iscrizione abusiva. Rilascio in tre fasi ordinate: **DB additivo → deploy client → lockdown RLS in transazione unica**. L'SQL è versionato in `db/` ma applicato a mano dal SQL editor del dashboard.

**Tech Stack:** Supabase (PostgreSQL + PostgREST + Gotrue), Blazor WebAssembly .NET 10, `supabase-csharp` (Postgrest/Gotrue), deploy GitHub Pages via GitHub Actions.

## Global Constraints

- Si lavora **solo sul branch `main`**; **push su main = deploy automatico in produzione** (GitHub Actions). Niente push alla cieca.
- **Commit/push solo su ok esplicito dell'utente** (incluso il commit dei doc).

- **Ordine di rilascio obbligatorio** (vincolo duro): (1) oggetti DB additivi → (2) deploy del nuovo client → (3) lockdown RLS. Invertire (3) prima di (2) rompe il join degli utenti live.

- Le RLS non sono unit-testabili in xUnit senza DB (manca l'interfaccia sui service — Fase 2): la verifica è **manuale** (run locale loggato + checklist a due account). Non si fabbricano test finti.

```sql
-- Oggetti ADDITIVI (retro-compatibili). Eseguire sul dashboard PRIMA del deploy client (Task 4).
create or replace function public.is_campaign_member(p_campaign uuid)
returns boolean language sql security definer stable set search_path = public as $$
  select exists (select 1 from campaign_members
                 where campaign_id = p_campaign and user_id = auth.uid());
$$;

create or replace function public.is_campaign_master(p_campaign uuid)
returns boolean language sql security definer stable set search_path = public as $$
  select exists (select 1 from campaign_members
                 where campaign_id = p_campaign and user_id = auth.uid() and role = 'master');
$$;

create or replace function public.can_read_character(p_char uuid)
returns boolean language sql security definer stable set search_path = public as $$
  select exists (select 1 from characters c
                 where c.id = p_char and public.is_campaign_member(c.campaign_id));
$$;

create or replace function public.can_edit_character(p_char uuid)
returns boolean language sql security definer stable set search_path = public as $$
  select exists (select 1 from characters c
                 where c.id = p_char
                   and (c.owner_id = auth.uid() or public.is_campaign_master(c.campaign_id)));
$$;

-- Solo SE l'audit (Step 6) ha mostrato duplicati: deduplicare tenendo la riga più vecchia.
-- delete from campaign_members a using campaign_members b
-- where a.ctid < b.ctid and a.campaign_id = b.campaign_id and a.user_id = b.user_id;

-- Vincolo unique (idempotente): salta se già presente dall'audit.
alter table campaign_members
  add constraint campaign_members_campaign_user_uniq unique (campaign_id, user_id);
```

### Task 3: Oggetto DB additivo — RPC `join_campaign`

Crea la funzione di join lato server. **Additiva**: il client attuale non la usa ancora, ma deve esistere prima del deploy del nuovo client (Task 4).

**Files:**
- Create: `db/20-join-campaign.sql`

**Interfaces:**
- Consumes: il vincolo `unique` (Task 2) per `on conflict`.
- Produces: `public.join_campaign(p_code text) → uuid` (id campagna se il codice è valido + membership `player` garantita; `null` se codice invalido). Consumato dal C# in Task 4.

- [ ] **Step 1: Creare `db/20-join-campaign.sql`**

```sql
-- Join campagna server-side (valida il codice, garantisce la membership player). ADDITIVO.
create or replace function public.join_campaign(p_code text)
returns uuid language plpgsql security definer set search_path = public as $$
declare v_campaign uuid;
begin
  select id into v_campaign from campaigns where invite_code = upper(trim(p_code));
  if v_campaign is null then return null; end if;           -- codice non valido
  insert into campaign_members (campaign_id, user_id, role, joined_at)
  values (v_campaign, auth.uid(), 'player', now())
  on conflict (campaign_id, user_id) do nothing;            -- già membro → no-op
  return v_campaign;
end; $$;
```

### Task 4: Client — `JoinCampaignAsync` passa a `join_campaign`

**Files:**
- Modify: `Services/SupabaseService.cs:456-493` (metodo `JoinCampaignAsync`)

**Interfaces:**
- Consumes: `public.join_campaign(p_code text) → uuid` (Task 3).
- Produces: comportamento invariato verso i chiamanti (`Pages` che chiamano `JoinCampaignAsync(inviteCode, userId)` → `Campaign?`).

- [ ] **Step 1: Sostituire il corpo di `JoinCampaignAsync`**

Rimpiazzare il metodo attuale (RPC `find_campaign_by_invite_code` + verifica + INSERT manuale) con la singola RPC `join_campaign`, poi la rilettura della campagna:

```csharp
public async Task<Campaign?> JoinCampaignAsync(string inviteCode, string userId)
{
    var client = await GetClientAsync();
    var code = inviteCode.Trim().ToUpperInvariant();

    // join_campaign (SECURITY DEFINER) valida il codice e garantisce la membership "player"
    // server-side, bypassando la RLS. Ritorna l'id campagna o null se il codice non è valido.
    // Sostituisce il vecchio flusso find_campaign_by_invite_code + INSERT diretto, che la nuova
    // policy campaign_members_insert (solo owner-come-master) bloccherebbe.
    var campaignId = await client.Rpc<string>(
        "join_campaign",
        new Dictionary<string, object> { { "p_code", code } });

    if (string.IsNullOrEmpty(campaignId)) return null; // codice non valido

    // Ora siamo membri: la RLS campaigns_select ci permette di leggere la campagna.
    var found = await client.From<Campaign>()
        .Where(c => c.Id == campaignId)
        .Get();
    return found.Models.FirstOrDefault();
}
```

```sql
begin;

-- characters
alter table characters enable row level security;
create policy characters_select on characters for select using ( is_campaign_member(campaign_id) );
create policy characters_insert on characters for insert
  with check ( owner_id = auth.uid() and is_campaign_member(campaign_id) );
create policy characters_update on characters for update
  using ( owner_id = auth.uid() or is_campaign_master(campaign_id) )
  with check ( owner_id = auth.uid() or is_campaign_master(campaign_id) );
-- niente DELETE (la cascata da campaigns bypassa le RLS)

-- notes
alter table notes enable row level security;
create policy notes_select on notes for select
  using ( owner_id = auth.uid() or (is_shared and is_campaign_member(campaign_id)) );
create policy notes_insert on notes for insert
  with check ( owner_id = auth.uid() and is_campaign_member(campaign_id) );
create policy notes_update on notes for update using ( owner_id = auth.uid() ) with check ( owner_id = auth.uid() );
create policy notes_delete on notes for delete using ( owner_id = auth.uid() );

-- inventory
alter table inventory enable row level security;
create policy inventory_select on inventory for select using ( can_read_character(character_id) );
create policy inventory_insert on inventory for insert with check ( can_edit_character(character_id) );
create policy inventory_update on inventory for update using ( can_edit_character(character_id) )
                                                       with check ( can_edit_character(character_id) );
create policy inventory_delete on inventory for delete using ( can_edit_character(character_id) );

-- character_spells
alter table character_spells enable row level security;
create policy cspells_select on character_spells for select using ( can_read_character(character_id) );
create policy cspells_insert on character_spells for insert with check ( can_edit_character(character_id) );
create policy cspells_update on character_spells for update using ( can_edit_character(character_id) )
                                                            with check ( can_edit_character(character_id) );
create policy cspells_delete on character_spells for delete using ( can_edit_character(character_id) );

-- cataloghi: spells / monsters / races / classes (stessa forma; added_by NULL ⇒ solo master modifica)
alter table spells enable row level security;
create policy spells_select on spells for select using ( is_campaign_member(campaign_id) );
create policy spells_insert on spells for insert with check ( added_by = auth.uid() and is_campaign_member(campaign_id) );
create policy spells_update on spells for update using ( added_by = auth.uid() or is_campaign_master(campaign_id) )
                                                 with check ( added_by = auth.uid() or is_campaign_master(campaign_id) );
create policy spells_delete on spells for delete using ( added_by = auth.uid() or is_campaign_master(campaign_id) );

alter table monsters enable row level security;
create policy monsters_select on monsters for select using ( is_campaign_member(campaign_id) );
create policy monsters_insert on monsters for insert with check ( added_by = auth.uid() and is_campaign_member(campaign_id) );
create policy monsters_update on monsters for update using ( added_by = auth.uid() or is_campaign_master(campaign_id) )
                                                     with check ( added_by = auth.uid() or is_campaign_master(campaign_id) );
create policy monsters_delete on monsters for delete using ( added_by = auth.uid() or is_campaign_master(campaign_id) );

alter table races enable row level security;
create policy races_select on races for select using ( is_campaign_member(campaign_id) );
create policy races_insert on races for insert with check ( added_by = auth.uid() and is_campaign_member(campaign_id) );
create policy races_update on races for update using ( added_by = auth.uid() or is_campaign_master(campaign_id) )
                                               with check ( added_by = auth.uid() or is_campaign_master(campaign_id) );
create policy races_delete on races for delete using ( added_by = auth.uid() or is_campaign_master(campaign_id) );

alter table classes enable row level security;
create policy classes_select on classes for select using ( is_campaign_member(campaign_id) );
create policy classes_insert on classes for insert with check ( added_by = auth.uid() and is_campaign_member(campaign_id) );
create policy classes_update on classes for update using ( added_by = auth.uid() or is_campaign_master(campaign_id) )
                                                 with check ( added_by = auth.uid() or is_campaign_master(campaign_id) );
create policy classes_delete on classes for delete using ( added_by = auth.uid() or is_campaign_master(campaign_id) );

-- combat_state (scrittura solo master; INSERT+UPDATE per l'Upsert)
alter table combat_state enable row level security;
create policy combat_select on combat_state for select using ( is_campaign_member(campaign_id) );
create policy combat_insert on combat_state for insert with check ( is_campaign_master(campaign_id) );
create policy combat_update on combat_state for update using ( is_campaign_master(campaign_id) )
                                                      with check ( is_campaign_master(campaign_id) );

-- campaigns (owner nella SELECT per non rompere il RETURNING dell'insert)
alter table campaigns enable row level security;
create policy campaigns_select on campaigns for select using ( owner_id = auth.uid() or is_campaign_member(id) );
create policy campaigns_insert on campaigns for insert with check ( owner_id = auth.uid() );
create policy campaigns_update on campaigns for update using ( owner_id = auth.uid() ) with check ( owner_id = auth.uid() );
create policy campaigns_delete on campaigns for delete using ( owner_id = auth.uid() );

create policy members_insert_owner on campaign_members for insert

  using ( user_id = auth.uid() or is_campaign_master(campaign_id) );

alter table profiles enable row level security;
create policy profiles_select on profiles for select using ( auth.uid() is not null );
create policy profiles_insert on profiles for insert with check ( id = auth.uid() );
create policy profiles_update on profiles for update using ( id = auth.uid() ) with check ( id = auth.uid() );
```
